プラットフォームパブリックポリシーには、system/sepolicy/publicに定義されているすべてのものが含まれます。
プラットフォームは、パブリックポリシーで定義されたタイプと属性が、特定のプラットフォームバージョンの安定したAPI
であると想定できます。 これは、非プラットフォーム（すなわち、デバイス）ポリシー開発者がデバイス固有の追加ポリシーを
書くことができる、プラットフォームによってエクスポートされるセポリポリシーの一部を形成します。

typeは、PLATFORM_SEPOLICY_VERSIONビルド変数で定義されている非プラットフォームファイルが書き込まれるポリシーの
バージョンに従ってバージョン管理されます。 バージョン管理された公開ポリシーは、非プラットフォームポリシーに含まれて
おり、プラットフォームポリシーに（元の形式で）含まれています。 

したがって、最終ポリシーには、
＞現在のプラットフォームのパブリックポリシー
＞プライベートプラットフォームポリシー
＞デバイス固有ポリシー
＞デバイスポリシーが書き込まれたプラットフォームバージョンに対応するバージョン管理されたパブリックポリシー
が含まれます。
＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝

よって、system/sepolicy/public/file.teに存在すべきtypeは

device/lineage/sepolicy/common/private
device/lineage/sepolicy/common/vendor
device/qcom/sepolicy-legacy/common
device/qcom/sepolicy-legacy/common/file.te
device/qcom/sepolicy-legacy/legacy-common
device/qcom/sepolicy-legacy/private
device/qcom/sepolicy/common
device/samsung/msm8974-common/sepolicy/common
system/sepolicy/private
system/sepolicy/vendor

から収集しておくこと。さもないと、unknown type aaaみたいなエラーが出る。 
(例) unknown type adsprpcd 
前提条件 
device/qcom/sepolicy-legacy/common/adsprpcd.te
と言うファイルの先頭部分は

# adsprpcd daemon 
type adsprpcd, domain; 
type adsprpcd_exec, exec_type, vendor_file_type, file_type; 
となっている。着目すべきは、type adsprpcd, domain;のみで

# adsprpcd daemon 
#type adsprpcd, domain; 
type adsprpcd_exec, exec_type, vendor_file_type, file_type; 

と先頭に＃を追加してコメントに変更する。
type adsprpcd, deamon 
自体は
system/sepolicy/public/file.te 
に引っ越しするとエラーが解決する。
